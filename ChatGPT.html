GitHub Pages

GitHub Pages est un service d’hébergement web statique fourni par GitHub, permettant aux utilisateurs de publier directement depuis un dépôt Git. Il est largement utilisé pour héberger des portfolios, documentations, blogs et sites de projets open source.

Faits clés

  • Lancement : 2008

  • Type : Hébergement de sites statiques

  • Propriétaire : GitHub (filiale de Microsoft)

  • Langages pris en charge : HTML, CSS, JavaScript, Jekyll

  • Dépôt source : N’importe quel dépôt GitHub public ou privé

Fonctionnement

GitHub Pages génère et diffuse des fichiers HTML statiques à partir d’un dépôt GitHub. Les utilisateurs peuvent choisir une branche (main, gh-pages) ou un dossier (/docs) comme source. Le service s’intègre avec Jekyll, permettant de transformer du contenu Markdown en site web sans serveur.

Intégration et automatisation

Grâce à GitHub Actions, les utilisateurs peuvent automatiser la construction et le déploiement de leurs sites. Cette intégration facilite la mise à jour continue du contenu, la génération dynamique de documentation, et le déploiement à partir de workflows CI/CD.

Utilisation et limites

GitHub Pages est gratuit pour les dépôts publics, avec des options restreintes pour les dépôts privés. Il prend en charge les domaines personnalisés et le HTTPS automatique via Let's Encrypt, mais n’autorise pas les scripts côté serveur ni les bases de données, car il est conçu pour les sites purement statiques.

Importance et adoption

GitHub Pages s’est imposé comme une solution de déploiement simple et intégrée pour la communauté open source. Il sert notamment à héberger la documentation de nombreux frameworks, bibliothèques et projets individuels, contribuant à la diffusion et la transparence du code.
